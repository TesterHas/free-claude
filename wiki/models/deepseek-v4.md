# DeepSeek V4

**Released:** Late April 2026
**Maker:** DeepSeek (Chinese AI lab)
**Key claim:** Up to 100x cheaper than Claude for equivalent tasks when used via API

## What it is

DeepSeek V4 is the latest model from DeepSeek that Jack Roberts shows can be combined with the Claude Code framework to dramatically cut costs. The core setup: use Claude Code's workflow and skills system, but route inference through DeepSeek V4's API instead of Claude's.

## Why it matters

- **100x cheaper** than Claude API pricing per token (Jack Roberts' claim)
- **No rate limitations** — you can build and run as much as you want
- **Fast setup** — Roberts shows a repo that lets you configure this in minutes
- Works even for complete beginners

## The setup

Roberts' approach: Claude Code framework + DeepSeek V4 as the backend model. You get:
- All the Claude Code UX and skills system
- DeepSeek V4's token costs
- No usage limits that would block heavy agentic workflows

## Trade-offs

- DeepSeek is not Anthropic — different model, different alignment, different privacy posture
- Performance may not match Claude Opus 4.7 on complex tasks
- Data sovereignty questions (Chinese lab) are relevant for some clients
- Not suitable for sensitive/regulated workloads

## Sales angle

For prospects who object to Claude/API costs: DeepSeek V4 is the cost objection answer. Build the same systems for a fraction of the cost. Then upgrade to Claude when the value is proven and budget is unlocked.
