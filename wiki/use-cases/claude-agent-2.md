# Claude Agent 2.0 / Memory System

**Source:** Jack Roberts — "I replaced Hermes... Claude Agent 2.0" (May 2026, 11 min) + "This New Claude Memory System = CHEAT CODE" (May 2026, 17 min)

## Claude Agent 2.0

Jack replaced "Hermes" (a previous agent framework he was using) with a new Claude-native agent setup he calls "Claude Agent 2.0."

Hermes was apparently an external agent framework for Claude. Claude's native agent capabilities have matured enough that the third-party wrapper is no longer necessary — Claude handles orchestration natively.

The implication: the agent ecosystem is consolidating around Claude's built-in features. External frameworks that added value 6–12 months ago are becoming redundant.

## The New Claude Memory System

Jack covers a new Claude memory approach as a "cheat code" — implying it delivers capabilities beyond what most users know about.

Based on the title and Jack's channel focus:
- Persistent memory across sessions
- Memory that updates and improves as you use Claude
- Potentially more sophisticated than simple CLAUDE.md files

This pairs with [[../new-skills/karpathy-skills]] as the two sides of long-term Claude intelligence: **skills** carry procedural memory, **memory files** carry factual/contextual memory.

## Why Agent 2.0 Matters for Selling

If Hermes → Claude Agent 2.0 is the pattern, it means:
- Less complexity in the stack (fewer third-party dependencies)
- More reliability (native features are better maintained)
- Easier client handoffs (standard Claude tools, not custom frameworks)

Simpler stacks sell better and support better. Native > third-party when the capability is equivalent.
