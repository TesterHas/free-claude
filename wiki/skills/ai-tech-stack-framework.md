# Overwhelmed By AI? Just Copy My Tech Stack

**Creator:** Nate Herk | AI Automation · 17:13 · [Watch](https://www.youtube.com/watch?v=35WuZxbAY68)
**Farmed:** 2026-05-08

A mindset and decision framework for managing AI tool overwhelm. Nate breaks his tech stack into tiers (S/A/B/C/Graduated) and shares the mental models he uses to decide what to adopt and what to skip.

## Nate's Tier System

| Tier | Tools | Frequency |
|------|-------|-----------|
| **S** (Daily drivers) | Claude Code, VS Code, Glido (speech-to-text) | Every day |
| **A** (Weekly) | Codex, Claude Chat, Hermes Agent, Perplexity, Groq | Weekly |
| **B** (Specialists) | Apify, GPT Image 2, Nano Banana 2, Kling.ai, OpenRouter, HeyGen, ElevenLabs, Claude Design | Per-task |
| **C** (Experimenting) | Gemini, Anti-Gravity, Ollama, Manifold | Occasional |
| **Graduated** | ChatGPT, OpenClaw, Cursor, NotebookLM, Poppy AI, Anytten, WhisperFlow | No longer used |

## Decision Framework

1. **North Star** — Know what you're optimizing for. "Every single new hot tool is it actually your path to the North Star or a distraction?"
2. **20% Productivity Dip Rule** — Every tool switch costs ~20% efficiency temporarily. Only switch if the post-dip trajectory exceeds your current ceiling.
3. **Needle Moved Per Hour, Not Hours Worked** — 4 focused hours > 12 scattered hours.
4. **Tool-Agnostic Directories** — "Build directories like they're going to outlive any tool — because they will." All agents can work on the same project directory.
5. **Task-Level Tool Selection** — Break processes into mini-tasks, pick the best tool per task. Not "which tool" but "for this specific task in this specific context, which tool is best."

## Pain Point Decision Flow

New tool/feature drops → Does it solve a pain point I have RIGHT NOW? → Yes: test in real scenario, evaluate after a week → No: save link, move on

## Core Stack (S+A tiers)

Only 6 tools: Claude Code, Glido, Codex, Hermes Agent, Perplexity, Groq. Everything else is either specialist or graduated.
