# Overwhelmed By AI? Just Copy My Tech Stack

**Channel:** Nate Herk | AI Automation  **URL:** https://www.youtube.com/watch?v=35WuZxbAY68  **Uploaded:** 2026-05-08  **Duration:** 17:13

## Summary

- Claude Code is Nate's S-tier daily driver and personal operating system; VS Code and Glydo (speech-to-text) round out his core stack
- A-tier tools include Codex (complements Claude Code), Claude Chat, Hermes Agent (Telegram-based), Perplexity, and Groq for agentic research
- Coding agents are 'just harnesses' — build your directories/projects tool-agnostically so any AI can plug in and your work outlives any specific tool
- Apply the '20% productivity dip rule': only switch tools when the temporary dip in efficiency will take you significantly higher, not just back to baseline
- Productivity = needle moved per hour, not hours worked — align every daily task tightly to your North Star goal before exploring new tools
- Decision framework for new AI tools: ask 'does this solve a current pain point?' If no, save the link; if yes, test in a real (not mock) scenario for one week
- Keep a lean core stack; use specialist tools (Apify, HeyGen, ElevenLabs, GPT-image-2) only for specific tasks within a process, mixing AI and non-AI steps freely
